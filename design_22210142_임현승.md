# 1. Introduction

## 1.1 Summary

카페에서 근무하는 아르바이트생과 신규 직원들은 다양한 메뉴의 레시피를 숙지해야 하며, 근무 시간과 급여 또한 스스로 관리해야 한다. 그러나 메뉴 수가 증가하고 신메뉴가 지속적으로 출시되면서 모든 레시피를 암기하는 것은 쉽지 않다. 또한 근무 시간을 수기로 기록하거나 급여를 직접 계산하는 과정에서 실수가 발생할 수 있다.

이러한 문제를 해결하기 위해 본 프로젝트에서는 **카페매니저+(CafeManager+)** 시스템을 개발한다. 카페매니저+는 메뉴 및 레시피를 효율적으로 관리하고, 사용자가 필요한 메뉴 정보를 빠르게 검색할 수 있도록 지원한다. 또한 즐겨찾기 기능을 통해 자주 사용하는 메뉴를 관리할 수 있으며, 근무 정보 입력 및 예상 급여 조회 기능을 제공하여 근무 관리의 편의성을 향상시킨다.

본 문서는 Analysis 단계에서 정의한 요구사항과 도메인 모델을 기반으로 작성된 Design 문서이다. Design 단계에서는 실제 구현에 필요한 클래스 구조와 객체 간 상호작용을 구체적으로 설계하며, Class Diagram, Sequence Diagram, State Machine Diagram을 통해 시스템의 정적 구조와 동적 동작을 정의한다. 또한 구현 단계에서 필요한 환경 및 기술적 요구사항을 제시하여 시스템 개발의 기준을 마련한다.

---

## 1.2 Design Goals

카페매니저+ 시스템의 설계 목표는 다음과 같다.

### 1) 효율적인 레시피 관리

사용자가 메뉴 및 레시피 정보를 쉽게 등록, 수정, 삭제할 수 있도록 설계한다. 또한 메뉴 검색 및 레시피 조회 기능을 제공하여 신규 직원도 필요한 정보를 빠르게 확인할 수 있도록 한다.

### 2) 사용자 중심의 정보 접근성 제공

자주 사용하는 메뉴를 즐겨찾기로 등록할 수 있도록 하여 반복적인 검색 과정을 최소화하고 사용자 편의성을 향상시킨다.

### 3) 정확한 근무 및 급여 관리

사용자가 근무 시간을 기록하고 예상 급여를 자동으로 계산할 수 있도록 설계하여 수작업 계산 과정에서 발생할 수 있는 오류를 줄인다.

### 4) 높은 유지보수성과 확장성 확보

시스템을 기능별 클래스로 분리하고 객체지향 설계 원칙을 적용하여 유지보수가 용이하도록 설계한다. 또한 향후 기능 추가 및 시스템 확장이 가능하도록 모듈화된 구조를 적용한다.

### 5) 구현과 설계의 일관성 유지

본 Design 문서에서 정의한 클래스, 메서드, 객체 간 관계 및 동작 흐름은 이후 Implementation 단계에서 실제 소스 코드 구조와 최대한 일치하도록 설계한다.

---

## 1.3 Important Points of Design

본 시스템 설계 시 다음 사항을 중요하게 고려하였다.

* 메뉴 및 레시피 관리 기능의 직관적인 사용성 제공
* 사용자 정보와 근무 정보의 안전한 관리
* 객체 간 책임 분리를 통한 높은 응집도와 낮은 결합도 유지
* Analysis 단계의 Use Case와 Design 단계의 UML 모델 간 일관성 유지
* 향후 기능 확장을 고려한 유연한 클래스 구조 설계
* Android 기반 애플리케이션 구현을 고려한 설계 구조 적용

본 Design 문서에서는 위 목표를 바탕으로 시스템의 Class Diagram, Sequence Diagram, State Machine Diagram을 설계하고 각 구성요소의 역할과 동작 과정을 구체적으로 기술한다.
